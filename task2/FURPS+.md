| Код | Требования                                                                                                                         | Комментарий                                              |
|-----|------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------|
| F   | Функциональные (Functionality)                                                                                                     |                                                          |
| F1  | Сайт должен отображать список доступных депозитов с актуальными ставками                                                           | Базовая функциональность для привлечения новых клиентов  |
| F2  | Сайт должен позволять новому клиенту подать заявку на депозит, указав ФИО и номер телефона                                         | Минимальный набор данных для обратной связи              |
| F3  | Интернет-банк должен отображать список доступных депозитов с актуальными ставками и персонализированными ставками для существующих | Персонализация на основе данных клиента в АБС            |
| F4  | Интернет-банк должен позволять клиенту подать заявку на открытие депозита с указанием счета и суммы                                | Клиент выбирает счет-источник средств                    |
| F5  | Интернет-банк должен требовать подтверждение операции через SMS-код                                                                | Соответствие требованиям безопасности и регуляторов      |
| F6  | Система должна обеспечивать передачу заявок с сайта в систему кол-центра                                                           | Интеграция для обработки заявок новых клиентов           |
| F7  | Система должна обеспечивать передачу заявок из интернет-банка в АБС для обработки бэк-офисом                                       | MVP                                                      |
| F8  | Система должна отправлять SMS-уведомления клиенту после: подтверждения ставки бэк-офисом и открытия депозита                       | Информирование клиента о статусах                        |
| F9  | Должна быть реализована функциональность управления ставками по депозитам (создание, изменение, расчет, согласование)              | Важная бизнес-логика, сейчас реализовано в Excel         |
| F10 | Сотрудники бэк-офиса (депозиты и кредиты) должны иметь возможность работать со ставками                                            | Разделение прав доступа - требование службы безопасности |
| U   | Удобство использования (Usability)                                                                                                 |                                                          |
| U1  | Интерфейсы сайта и интернет-банка должны соответствовать системе дизайна банка с использованием брендированных элементов           | Eдиный пользовательский опыт (UI/UX)                     |
| U2  | Интерфейс управления ставками для сотрудников бэк-офиса должен быть удобным и минимизировать ручной ввод                           | Снижение операционных ошибок                             |
| R   | Надёжность (Reliability)                                                                                                           |                                                          |
| R1  | Система должна быть доступна 99,9% времени                                                                                         | SLA                                                      |
| R2  | В случае сбоя в основном ЦОД должен обеспечиваться автоматический или ручной переход на резервный ЦОД                              |                                                          |
| R3  | Система должна обеспечивать сохранность данных при сбоях                                                                           |                                                          |
| P   | Производительность (Performa[]()nce)                                                                                               |                                                          |
| P1  | Система должна выдерживать пиковые нагрузки от онлайн-заявок без угрозы работоспособности АБС                                      |                                                          |
| P2  | Отклик сайта и интернет-банка на все операции должен быть в пределах 300-500 миллисекунд, особенно загрузка справочных данных      | Требование по производительности                         |
| P3  | Интернет-банк должен поддерживать горизонтальное масштабирование для распределения нагрузки                                        | Для обеспечения отказоустойчивости и производительности  |
| P4  | АБС может масштабироваться только вертикально, что должно учитываться в архитектуре                                                | Ограничение АБС                                          |
| P5  | Система управления ставками должна обеспечивать быстрый расчет персонализированных ставок                                          | Для онлайн-отображения в интернет-банке                  |
| S   | Поддерживаемость (Supportability)                                                                                                  |                                                          |
| S1  | Система должна быть документирована для дальнейшего расширения                                                                     |                                                          |
| S2  | По возможности использовать уже имеющиеся в банке технологии и по которым уже есть экспертиза                                      | Снижение рисков                                          |
| S3  | Новые технологии должны быть совместимы с существующими платформами                                                                |                                                          |
| S4  | Архитектура должна позволять постепенный переход на микросервисную архитектуру для интернет-банка                                  | Стратегическое требование                                |
| +R  | + Ограничения (Restrictions)                                                                                                       |                                                          |
| +R1 | Нельзя напрямую обращаться к АБС из интернет-банка в новом процессе                                                                | Требование по защите от перегрузки АБС                   |
| +R2 | Необходимо шифрование трафика для передачи чувствительных данных (сайт и интернет-банк)                                            | Требование безопасности                                  |
| +R3 | Платформа интернет-банка (текущая версия) несовместима с Kafka                                                                     | Техническое ограничение                                  |
| +R4 | База данных АБС (Oracle) перегружена, нужно минимизировать нагрузку                                                                | Критическое ограничение                                  |
